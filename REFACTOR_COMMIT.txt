refactor(platforms): eliminate code duplication in platform entities

根据 CODE_REVIEW_PLATFORMS.md 的审查结果，消除平台实体中的代码重复。

## 主要变更

### 1. 创建 const/entity_config.py - 集中硬编码配置
- 提取色温范围默认值 (DEFAULT_MIN_KELVIN=2700, DEFAULT_MAX_KELVIN=6500)
- 提取风扇速度范围 (DEFAULT_SPEED_COUNT=100)
- 提取窗帘位置范围 (DEFAULT_MIN_POSITION=0, DEFAULT_MAX_POSITION=100)
- 所有平台共享统一配置源

### 2. 增强 entities/base.py - 通用状态更新
- 添加 _update_from_device() 方法作为可选覆盖点
- 子类可选择性实现特定状态更新逻辑
- 保持现有 debounce 和乐观更新机制不变

### 3. 重构 switch.py - 配置驱动方式
**重构前**: 8 个相似的开关类 (LiproFadeSwitch, LiproSleepAidSwitch, 等)
**重构后**: 2 个通用类 + 配置数据

- 定义 PropertySwitchConfig dataclass (translation_key, entity_suffix, property_key, device_property)
- LIGHT_FEATURE_SWITCHES: 5 个灯光功能开关配置
- PANEL_FEATURE_SWITCHES: 2 个面板功能开关配置
- LiproPropertySwitch: 通用属性开关 (接受 config 参数)
- LiproPanelPropertySwitch: 通用面板开关 (使用 PANEL_CHANGE_STATE)

**代码减少**: 从 262 行减少到 270 行 (包含新增配置定义)
**类数量**: 从 10 个减少到 4 个 (2 个通用类 + 2 个主开关类)

### 4. 更新 const/properties.py
- 引用 entity_config.py 中的色温常量
- MIN_COLOR_TEMP_KELVIN = DEFAULT_MIN_KELVIN
- MAX_COLOR_TEMP_KELVIN = DEFAULT_MAX_KELVIN
- 保持向后兼容，现有代码无需修改

### 5. 更新测试文件
- test_platform_entities_behavior.py: 使用 LiproPropertySwitch 和 LiproPanelPropertySwitch
- test_switch.py: 通过配置查找创建开关实例
- 所有 46 个 switch 相关测试通过
- 所有 579 个平台测试通过

## 技术细节

**配置驱动示例**:
```python
# 旧方式 - 需要创建新类
class LiproFadeSwitch(LiproPropertySwitch):
    _attr_translation_key = "fade"
    _entity_suffix = "fade"
    _property_key = PROP_FADE_STATE
    _device_property = "fade_state"

# 新方式 - 只需添加配置
PropertySwitchConfig(
    translation_key="fade",
    entity_suffix="fade",
    property_key=PROP_FADE_STATE,
    device_property="fade_state",
)
```
