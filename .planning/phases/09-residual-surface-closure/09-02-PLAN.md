<objective>
封住 runtime live mutable dict 泄露，并把 outlet power 的正式读写路径从 `extra_data` 旁写迁移到单一 formal primitive，使 coordinator / sensor / diagnostics / helpers 共用同一 truth。
</objective>

<tasks>
<task>为 `Coordinator` 设计并落地 read-only device access contract：优先使用 read-only mapping 或正式 service/view，确保平台、helpers、diagnostics 等现有读取方仍可遍历/索引设备，但不能经 public property 直接改写 runtime registry。</task>
<task>迁移所有直接依赖 `coordinator.devices` live mutable 语义的生产 consumers（如 platform helpers、diagnostics 读取路径）到新的 formal read surface，并为现有 public behavior 补充 targeted regression tests。</task>
<task>在 `custom_components/lipro/core/coordinator/outlet_power.py` 与相关 device/runtime home 中建立 outlet power formal primitive，切换 coordinator 写路径、sensor/diagnostics 读路径；如短期需桥接旧 `extra_data["power_info"]` 读取，必须把桥接限定为过渡兼容层并标注 delete gate。</task>
<task>同步更新 `PUBLIC_SURFACES.md` 与 `AUTHORITY_MATRIX.md`（如 power truth home 变化），并通过 `tests/core/test_outlet_power.py`、`tests/platforms/test_sensor.py`、`tests/core/test_diagnostics.py`、`tests/test_coordinator_public.py` 证明 runtime/device/power truth 已收口。</task>
</tasks>

## Verification Criteria

- `Coordinator` public contract 不再暴露 live mutable dict；
- 既有生产 consumers 仍能通过 formal read surface 读取设备；
- outlet power 写入与读取共享单一 primitive；
- `extra_data` 不再承担正式 outlet power truth 角色。

## Must Haves

- read-only runtime device access
- one formal outlet-power primitive
- consumers migrated to the same truth
- no live mutable registry leak through public API
