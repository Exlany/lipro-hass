## Verification Criteria

- Python 版本、Ruff target、pre-commit 默认解释器、devcontainer 与 CI 使用同一条版本故事，不再出现 `3.14` / `py313` / `python3.13` 混搭；
- `pytest` markers 只保留真实使用且有意义的 contract，或被明确删除，不再制造“看似支持切片、实际无人消费”的假象；
- `scripts/develop` 与 `CONTRIBUTING.md` 对本地开发步骤、依赖准备、启动方式和 canonical 命令的描述完全一致；
- `scripts/develop` 的非破坏性有显式 smoke 或 manual gate，不再只靠 `shellcheck` 假定安全。

## Must-Haves

- Python / Ruff / pre-commit / devcontainer / CI 只有一个正式版本真相
- 不保留死 marker、死声明或“绿色但不可解释”的工具契约
- `scripts/develop` 必须是诚实的本地入口，而不是 undocumented side path
- contributor-facing DX 仍以 `uv run ...` 与现有 CI 主线为准，不另起第二工作流
