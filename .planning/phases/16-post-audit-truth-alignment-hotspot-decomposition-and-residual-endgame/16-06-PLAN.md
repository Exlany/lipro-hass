## Verification Criteria

- 平台测试的职责边界更清晰：adapter/projection/wiring 留在 `tests/platforms/*`，领域语义与 OTA 策略不再继续在平台层双重驻留；
- `docs/README.md`、`CONTRIBUTING.md`、`README.md`、`README_zh.md` 能把读者可靠导向 troubleshooting、release runbook 与正式治理入口；
- `SUPPORT.md`、`SECURITY.md`、issue/PR template 与 `release.yml` 对单维护者 review / release 模式使用同一条可执行叙事；
- second-pass repo audit 证明每个 remaining residual / fallback / broad-catch / debt hotspot 都已清理或被显式降格，不存在未登记的高风险 carry-forward。

## Must-Haves

- 平台测试只验证平台 adapter，不长期背负领域/OTA 策略主覆盖
- troubleshooting / runbook / contributor navigation 必须互相链接且路径唯一
- release 必须继续复用 `ci.yml` 的治理门禁，不得出现旁路发布故事线
- 不允许留下无 owner / 无 delete gate / 无风险说明的 high-risk carry-forward
