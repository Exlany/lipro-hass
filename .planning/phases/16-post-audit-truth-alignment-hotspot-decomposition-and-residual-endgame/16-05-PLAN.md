## Verification Criteria

- `LiproDevice` 的显式 surface 更聚焦，平台与实体不再依赖一串不断膨胀的 convenience passthrough；
- capability consumption story 更单一，平台过滤、descriptor 读取与领域 snapshot 不再长期并存两套近似语义；
- `firmware_update.py` 热点下降并回到 projection/action-bridge 角色，manifest/cache/arbitration/policy 逻辑更多归位到 `core/ota/*`；
- 本 plan 覆盖的热点文件必须记录 before/after 指标，且 `Any` / `except Exception` / 行数不得净增；若未下降，必须有显式、可验证的结构性理由。

## Must-Haves

- `LiproDevice` 不能继续长成第二套 public surface
- `firmware_update.py` 必须保持 projection + action bridge 身份
- firmware support 的 local trust root 与现有 arbitration 规则不得被削弱
- 不把平台特有策略逻辑反向塞回 domain root 或 entity root
