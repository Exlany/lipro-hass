## Verification Criteria

- `client.py`、`facade.py`、`coordinator.py` 的职责进一步聚焦，strategy / exception mapping / compat glue 不再继续在 root/façade 内膨胀；
- live seams 的 `Any` / `cast` / 反射口显著下降，类型收窄体现在真实 contract 上，而不是仅靠 `cast` 换皮；
- broad catch 只保留在明确的 circuit-breaker / shutdown / observability 位置，其余路径要么收窄异常类型，要么写成可判定的 arbitration；
- `_Client*`、`LiproMqttClient`、`get_auth_data()` fallback 与 compat envelope 的 residual story 被进一步本地化，没有开启 physical rename campaign 或第二条正式主链。

## Must-Haves

- `LiproProtocolFacade` 仍是唯一正式 protocol root
- 不允许把 residual cleanup 变成大规模 rename 或 second-root promotion
- raw vendor shape 不得借 hotspot 重构回流到 runtime / domain / control
- residual / public-surface / delete-gate 文档必须与代码真相同步，而不是继续靠历史描述兜底
