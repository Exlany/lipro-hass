## Verification Criteria

- `client.py`、`facade.py`、`coordinator.py` 的职责进一步聚焦，strategy / exception mapping / compat glue 不再继续在 root/façade 内膨胀；
- `request_policy.py`、`auth_service.py`、`status_fallback.py`、`rest_decoder.py`、`protocol/contracts.py`、`mqtt_runtime.py` 与其协作者不再作为“计划外热点”继续漂浮在正式主链上；
- live seams 的 `Any` / `type: ignore` / `cast` / 反射口显著下降，类型收窄体现在真实 contract 上，而不是仅靠 `cast` 换皮；
- 本 plan 覆盖的热点文件必须记录 before/after 指标，且 `Any` / `except Exception` / `type: ignore` / 行数不得净增；若未下降，必须有显式、可验证的结构性理由。

## Must-Haves

- `LiproProtocolFacade` 仍是唯一正式 protocol root
- 不允许把 residual cleanup 变成大规模 rename 或 second-root promotion
- raw vendor shape 不得借 hotspot 重构回流到 runtime / domain / control
- residual / public-surface / dependency / delete-gate 文档必须与代码真相同步，而不是继续靠历史描述兜底
