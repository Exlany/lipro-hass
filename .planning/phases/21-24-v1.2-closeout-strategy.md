# Phase 21-24 Strategy

**Milestone:** `v1.2 Host-Neutral Core & Replay Completion`
**Date:** `2026-03-16`
**Status:** `planned`

## Why A Four-Phase Closeout

`Phase 20` 已把 remaining boundary families 收口到 formal truth，但 `v1.2` 若想以“大师级收口”结束，后半程不能再把 replay、exception、observability、governance、release closeout 混在一个大篮子里执行。`21-24` 需要形成一条严格依赖链：

1. 先把 replay / evidence / exception taxonomy 变成可信技术底座；
2. 再把 failure taxonomy 暴露到 diagnostics / system health / support / developer consumers；
3. 再统一 governance / docs / templates / release evidence；
4. 最后做 final repo audit、archive readiness 与 v1.3 handoff。

## Cross-Phase Contract

- `21` 只负责 **replay/evidence 扩面 + broad-catch 收窄 + failure taxonomy 契约化**。
- `22` 只负责 **observability consumer convergence**，不提前写 milestone closeout。 
- `23` 只负责 **governance truth / contributor docs / release evidence** 同步。
- `24` 只负责 **final audit / residual arbitration / archive-ready closeout / v1.3 handoff**。

- v1.2 没有 remaining boundary-family drift。
- replay/evidence/observability/failure taxonomy 对内对外都只讲一条故事。
- baseline / reviews / docs / templates / release gates / closeout assets 全部对齐。
- residual ledger 中不再残留 silent defer；未做项必须明确递延到 v1.3。
