<tasks>

<task>为每条规则补齐 `rule_id / source_refs / governed_paths / allowed_or_forbidden / enforcement_kind / residual_gate / future_extension`，避免 policy 只剩口号化叙事。</task>
<task>同步回写 `DEPENDENCY_MATRIX / PUBLIC_SURFACES / AUTHORITY_MATRIX / VERIFICATION_MATRIX`，让现有 baseline 明确指向 policy spec，而不是继续各写各的 seed 文案。</task>
<task>裁决哪些规则属于 baseline 派生真源，哪些只是已知回归禁令；前者必须进入 policy baseline，后者必须明确标记为 targeted regression ban。</task>
<task>为 `07.3 telemetry exporter` 与 `07.4 replay harness` 预留 extension hooks，但不越权提前冻结它们的实现细节或 schema 细节。</task>
</tasks>

## Verification Criteria

- `ARCHITECTURE_POLICY.md` 能回答：每条规则来自哪份真源、约束谁、由什么门禁执行；
- `boundary-local` 与 `raw-payload` 规则已显式区分“内部协作者”与“正式 public surface”；
- compat / residual 例外不再隐含散落，而有 delete gate；
- `07.3 / 07.4` 能在现有 taxonomy 下扩展，不需要再造第二套 enforcement 叙事。

## Must Haves

- baseline-first single truth
- stable rule ids and source refs
- targeted bans separated from structural rules
- forward-compatible extension hooks
