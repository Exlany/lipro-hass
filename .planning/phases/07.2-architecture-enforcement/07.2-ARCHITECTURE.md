# Phase 7.2 Architecture

## Architectural Goal

把 v1.1 的 boundary / exporter / replay 演进线纳入一条更强的 architecture enforcement 主链，使北极星约束不只存在于文档叙事里，而能以统一规则在本地与 CI 自动仲裁。

## Single Truth Chain

```text
NORTH_STAR_TARGET_ARCHITECTURE.md
  -> baseline docs
     - TARGET_TOPOLOGY.md
     - DEPENDENCY_MATRIX.md
     - PUBLIC_SURFACES.md
     - AUTHORITY_MATRIX.md
     - VERIFICATION_MATRIX.md
     - ARCHITECTURE_POLICY.md   # Phase 7.2 new formal baseline
  -> executable policy helpers
     - tests/helpers/architecture_policy.py
     - tests/helpers/ast_guard_utils.py
  -> enforcement entrypoints
     - scripts/check_architecture_policy.py
     - tests/meta/test_dependency_guards.py
     - tests/meta/test_public_surface_guards.py
     - tests/meta/test_governance_guards.py
  -> CI governance job
```

裁决：
- baseline docs 是正式真源；
- helper/script 只是执行层；
- CI 只是 fail-fast 仲裁层；
- 不允许再出现“文档一套、测试一套、脚本又一套”的三口径分裂。

## Formal Components

### 1. `ARCHITECTURE_POLICY.md`

正式职责：
- 把 baseline truth 组织成稳定 rule taxonomy；
- 给每条规则分配 `rule_id`、`source_refs`、`scope`、`enforcement_kind`；
- 说明哪些规则是结构派生、哪些是已知回归禁令、哪些仍属于 future extension。

### 2. `tests/helpers/architecture_policy.py`

正式职责：
- 从 baseline docs 读取 rule tables / source refs；
- 暴露统一 policy query API；
- 为 tests/scripts 提供同一份规则投影。

### 3. `tests/helpers/ast_guard_utils.py`

正式职责：
- 收敛 AST 解析与 import/property/export 检测；
- 避免 guard 工具逻辑散落在多个测试文件中；
- 使 dependency/public-surface guards 更可组合。

### 4. `scripts/check_architecture_policy.py`

正式职责：
- 以脚本方式快速执行 plane/root/surface/authority 规则；
- 成为 developer-local 与 CI-governance 的第一道 fail-fast 门禁；
- 不承载 file inventory / active-doc drift 检查。

### 5. Focused meta guard shells

- `tests/meta/test_dependency_guards.py`
- `tests/meta/test_public_surface_guards.py`
- `tests/meta/test_governance_guards.py`

正式职责：
- 针对最容易回归的结构断点保留 focused assertions；
- 在 shared policy helper 之上执行更清晰的 regression proof；
- 不继续演化成各自维护一套独立真相。

## Rule Taxonomy

| Taxonomy | What it governs | Primary sources | Enforcement shape |
|----------|------------------|-----------------|-------------------|
| Plane | 五平面依赖方向 | `TARGET_TOPOLOGY.md`, `DEPENDENCY_MATRIX.md` | policy script + dependency guards |
| Root | 唯一正式根 / child façade 边界 | `NORTH_STAR_TARGET_ARCHITECTURE.md`, `PUBLIC_SURFACES.md` | policy script + public-surface guards |
| Surface | canonical / transitional / forbidden 三态 | `PUBLIC_SURFACES.md` | policy script + export/public-surface guards |
| Boundary-local | `core/protocol/boundary/*` 仅限 protocol/assurance 合法消费 | `PUBLIC_SURFACES.md`, `AUTHORITY_MATRIX.md` | dependency guards + authority-linked checks |
| Raw-payload | raw vendor payload 不得穿透 boundary | `NORTH_STAR_TARGET_ARCHITECTURE.md`, `VERIFICATION_MATRIX.md` | targeted guards + future replay proof |
| Compat / residual | compat 只能显式、可计数、可删除 | `PUBLIC_SURFACES.md`, `RESIDUAL_LEDGER.md`, `KILL_LIST.md` | policy script + governance drift checks |
| Authority | truth-source 顺序与 owner 归属 | `AUTHORITY_MATRIX.md`, `VERIFICATION_MATRIX.md` | policy script + governance guards |
| Backdoor | runtime internals / private seams / bypass surface | `PUBLIC_SURFACES.md`, residual docs | targeted denylist + focused AST checks |

## Extension Hooks For Future Phases

### `07.3` Telemetry exporter

`07.2` 必须预留：
- `observer-only components`
- `telemetry truth consumers`
- `redaction-policy-governed surfaces`

裁决：
- exporter 只能消费正式 telemetry truth；
- 不得获得 runtime 编排权；
- 不得直连 boundary decoder internals 或 raw payload。

### `07.4` Replay harness

`07.2` 必须预留：
- `assurance-only boundary consumers`
- `replay evidence families`
- `deterministic proof hooks`

裁决：
- replay harness 只能复用正式 façade/decoder path；
- 不得复制第二套 protocol implementation；
- 若确需直接接 boundary family，必须被 authority / verification 显式登记。

## Execution Shape

### `07.2-01`

先定义 policy baseline：
- 新增 `ARCHITECTURE_POLICY.md`
- 同步 dependency/public-surface/authority/verification baseline
- 把 rule taxonomy 与 extension hooks 正式落表

### `07.2-02`

再接 enforcement chain：
- shared policy loader
- shared AST helper
- fail-fast architecture script
- focused meta guard refactor
- governance CI 顺序升级

## Non-Goals

- 不在 07.2 引入独立规则引擎或新的运行时依赖
- 不在 07.2 一次性完成全仓治理 closeout（那是 07.5 的职责）
- 不在 07.2 越权定义 07.3 telemetry schema / 07.4 replay API 细节
