# Phase 7.2 Context

**Phase:** `7.2 Architecture Enforcement 加固`
**Milestone:** `v1.1 Protocol Fidelity & Operability`
**Date:** 2026-03-13
**Status:** Ready for planning
**Source:** Roadmap + Requirements + Phase 7.1 closeout

## User / Product Context

- `Phase 7.1` 已完成 protocol-boundary family、representative REST/MQTT decoder pipeline、replay-ready fixtures 与治理 handoff。
- 下一步目标不是继续扩功能，而是把这些新增结构提升为**不可轻易回退的架构规则**，让后续 `telemetry exporter`、`replay harness` 与治理收尾都建立在更强 enforcement 之上。
- 用户要求继续沿着北极星单一主链推进，不接受“规则只写在文档里、代码可随意偏航”的宽松状态。

## Architectural Context

- `LiproProtocolFacade` 仍是唯一正式 protocol root。
- `Coordinator` 仍是唯一正式 runtime root。
- `core/protocol/boundary/*` 已成为 decoder family formal home，但目前 guard 仍以 seed-level AST/import 检查为主。
- 当前 baseline truth 已明确存在于：
  - `.planning/baseline/DEPENDENCY_MATRIX.md`
  - `.planning/baseline/PUBLIC_SURFACES.md`
  - `.planning/baseline/AUTHORITY_MATRIX.md`
  - `.planning/baseline/VERIFICATION_MATRIX.md`
  - `.planning/reviews/FILE_MATRIX.md`
  - `.planning/reviews/RESIDUAL_LEDGER.md`
- `Phase 7.2` 必须把这些文档真源转化为更强的 architecture policy / meta guard / CI gate，而不是另造平行规则来源。

## Phase Goal

把 v1.1 新增 boundary/exporter/replay 组件纳入更强 architecture policy，形成可本地快速执行、可在 CI fail-fast 的 enforcement 体系，重点阻止：

1. 双主链回流；
2. boundary decoder internals 泄漏到 runtime/domain/control/entity；
3. compat shell / raw payload / bypass surface 的重新合法化；
4. authority / public surface / dependency truth 与代码实际脱节。

## Specific Inputs From Phase 7.1

- `07.1-01-SUMMARY.md`：已建立 boundary inventory、registry/result skeleton、authority/file-matrix sync。
- `07.1-02-SUMMARY.md`：已建立 representative REST/MQTT decoder pipeline，并新增 boundary leakage guards。
- `07.1-03-SUMMARY.md`：已建立 replay-ready fixture family，并登记 `Protocol-boundary family coverage` residual。
- 当前已存在的 guard seeds：
  - `tests/meta/test_dependency_guards.py`
  - `tests/meta/test_public_surface_guards.py`

  - `scripts/check_file_matrix.py`

## Non-Negotiable Constraints

- 不得引入第二 protocol root、第二 runtime root、第二 governance truth。
- 不得把 `core/protocol/boundary/*` 升格为 protocol plane 之外的 public surface。
- enforcement 必须首先来自 baseline docs，而不是 hardcode 一堆一次性字符串规则后再让文档倒着适配代码。
- 本地快速校验链必须存在；不能只依赖 CI 末端发现问题。
- 新 guard 若无法解释 owner / authority / public surface / dependency truth，就不算达标。

## Planning Expectations

- `07.2-01` 应优先建立 architecture policy spec 与 rule taxonomy，明确从哪些真源派生规则、哪些规则最先落地。
- `07.2-02` 应把 policy 落到 meta guards / CI gates / developer workflow 中，并明确最小充分验证命令。
- plans 必须显式写出：
  - 改哪些基线文档
  - 改哪些 guard/tests/scripts
  - 如何证明 local-fast + CI-fail-fast
  - 如何防止与 `07.3/07.4` 未来新增组件脱节
