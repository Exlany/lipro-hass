<objective>
继续收窄 `core` formal public surface 与 HA runtime root 的叙事边界：让 host-neutral core truth 聚焦 protocol/domain/auth/contracts，而 `Coordinator` 等 HA runtime root 明确留在 `coordinator_entry.py` / runtime public home，不再混作“可复用 core”的一部分。
</objective>

<tasks>
<task>审查 `core/__init__.py`、`coordinator_entry.py`、`runtime_types.py`、`control/runtime_access.py`、`control/telemetry_surface.py` 的当前 public-surface / adapter 依赖关系，明确哪些属于 host-neutral truth，哪些属于 HA runtime home。</task>
<task>收窄 `core` 顶层 formal exports，并把 `Coordinator` / runtime home 的正式叙事固定在 `coordinator_entry.py` 与 runtime public surface；必要兼容只能保留为显式、可计数、可删除的 transitional seam。</task>
<task>同步调整 control adapter（如 `runtime_access`、`telemetry_surface`）对 runtime/protocol concrete shape 的读取方式，优先改走 formal runtime/public surface 或 protocol snapshot contract。</task>
<task>回写 `PUBLIC_SURFACES.md`、`DEPENDENCY_MATRIX.md`、`ARCHITECTURE_POLICY.md` 与对应 meta guards，阻断 `core` 重新长回 HA runtime root 叙事。</task>
</tasks>

## Verification Criteria

- `Coordinator` 的正式 home 继续明确，不再被 `core` 顶层 public surface 混淆；
- control adapters 对 concrete runtime/protocol shape 的依赖减少；
- public-surface / dependency guards 能阻止 runtime home 回流到 host-neutral core truth；
- 现有 HA 主链不被破坏。

## Must Haves

- core truth vs HA runtime home is explicit
- coordinator exposure stays localized
- control adapters use formal surfaces
- guard chain reflects the new boundary
