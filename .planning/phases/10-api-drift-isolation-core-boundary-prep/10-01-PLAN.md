<objective>
把 `login`、`device_list`、`query_device_status`、`query_mesh_group_status`、OTA/support-critical payload 等高漂移输入继续收口到 protocol boundary/canonical contract，阻断 runtime/domain/control 对 vendor envelope、field alias、pagination 形态的直接理解。
</objective>

<tasks>
<task>梳理高漂移 endpoint 的现状矩阵：列出哪些调用仍返回或透传 `data` / `hasMore` / `iotId` / `deviceId` / `properties` / envelope code 等 vendor shape，明确 canonical contract 应落在哪个 protocol home。</task>
<task>在 `core/protocol/contracts.py`、`core/protocol/boundary/rest_decoder.py` 与相关 endpoint collaborators 中补齐或重构 canonical contracts / decoder helpers，让高漂移 endpoint 先在 boundary 内完成 canonicalization，再通过 `LiproProtocolFacade` 暴露正式结果。</task>
<task>迁移 `core/coordinator/orchestrator.py`、`core/coordinator/runtime/device/snapshot.py` 等 runtime 消费者，移除对 vendor envelope / alias / pagination 的理解，只消费新的 canonical contract 或显式 protocol helper。</task>
<task>为新增/变更 contract 补 replay / contract / regression proof：更新 `tests/fixtures/api_contracts/**`、`tests/fixtures/protocol_replay/**`、`tests/core/api/test_protocol_contract_matrix.py`、`tests/core/api/test_protocol_replay_rest.py`、`tests/core/mqtt/test_protocol_replay_mqtt.py`，确保 API 漂移优先打在 boundary。</task>
</tasks>

## Verification Criteria

- runtime/domain/control 不再自己解析高漂移 vendor envelope / field alias / pagination；
- `LiproProtocolFacade` / protocol helpers 为这些 endpoint 提供稳定 canonical results；
- replay / fixture / authority proof 覆盖新增或收口的 boundary families；
- API 漂移首先表现为 boundary/contract failure，而不是 HA adapter failure。

## Must Haves

- high-drift endpoint inventory is explicit
- canonical contracts live in protocol plane
- runtime stops understanding vendor envelopes
- replay and contract evidence updated
