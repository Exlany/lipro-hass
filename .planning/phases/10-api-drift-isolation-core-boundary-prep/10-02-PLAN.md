<objective>
把 `config_flow`、`entry_auth` 与相关 control adapters 从 raw login/result dict shape 中解耦，改为依赖 formal auth/session/result contract 或显式 use case，避免底层 API 改动直接打穿 HA adapter。
</objective>

<tasks>
<task>盘点 `config_flow.py`、`entry_auth.py`、control surfaces 中仍直接依赖 protocol login/result shape 的路径，明确哪些字段应上收为 formal auth/result contract，哪些逻辑应下沉为显式 use case。</task>
<task>在 `core/auth` / `core/protocol/contracts.py` 或其正式 home 中定义 host-neutral auth/session/result contracts，并为 login / refresh / reauth / entry bootstrap 提供稳定返回语义与错误映射边界。</task>
<task>迁移 HA adapters：`config_flow.py`、`entry_auth.py` 与相关 control code 改走 formal contract / use case，移除对 raw response dict key、临时 envelope shape 与 protocol private details 的直接依赖。</task>
<task>补 flow/auth targeted regression，证明登录、重认证、重新配置与 entry setup 在 contract 稳定时不受底层 payload 形态变化影响。</task>
</tasks>

## Verification Criteria

- `config_flow` / `entry_auth` 不再直接消费 raw login/result dict shape；
- auth/session/result contract 有清晰 formal home 与错误语义；
- flow/auth 回归继续通过；
- 未来 CLI / other host 若复用 auth use case，不必复制 HA-specific parsing。

## Must Haves

- auth/result contracts are explicit
- control adapters no longer parse vendor payloads
- reauth and reconfigure paths remain stable
- future host reuse story improves without second root
