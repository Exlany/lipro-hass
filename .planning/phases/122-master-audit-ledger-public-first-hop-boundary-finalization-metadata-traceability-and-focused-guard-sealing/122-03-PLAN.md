# Plan 122-03: 补齐 tagged-release metadata traceability 并封口 focused guards

## Scope

- 收口 release-facing metadata URL 与 semver/tagged-release truth 的对应关系
- 明确 `hacs.json` 只继续承载 Home Assistant minimum-version truth，不伪装成 release provenance 真源
- 用 focused guards 冻结 ledger completeness、public first-hop / appendix boundary、metadata tag traceability 与 active route truth

## Files

- `pyproject.toml`
- `custom_components/lipro/manifest.json`
- `hacs.json`
- `docs/README.md`
- `SUPPORT.md`
- `SECURITY.md`
- `tests/meta/test_version_sync.py`
- `tests/meta/test_phase75_access_mode_honesty_guards.py`
- `tests/meta/toolchain_truth_docs_fast_path.py`
- `tests/meta/test_phase122_master_audit_guards.py`（新建）

## Implementation Steps

1. 清点所有 release-facing metadata projection：`pyproject.toml::project.urls`、`custom_components/lipro/manifest.json::{documentation,issue_tracker}`、以及 docs 中公开声明的 provenance 语句，并统一锚定到 `v{project.version}`。
2. 校正 provenance 语义：拒绝 `main`、generic、stale 或 milestone-labeled target；`hacs.json` 继续只承载 Home Assistant minimum-version truth，不新增第二套 URL 真源。
3. 扩展 `tests/meta/test_version_sync.py`、`tests/meta/test_phase75_access_mode_honesty_guards.py`、`tests/meta/toolchain_truth_docs_fast_path.py`，确保既校验路径后缀，也校验 `/blob/v<version>/` tag 语义。
4. 新建 `tests/meta/test_phase122_master_audit_guards.py`，把 `v1.35` 专属 contract（ledger completeness、public/appendix 边界、route truth、metadata traceability）固定在一个 focused home，而不是继续散落在口头结论里。

## Validation

- `uv run pytest tests/meta/test_version_sync.py tests/meta/test_phase75_access_mode_honesty_guards.py tests/meta/toolchain_truth_docs_fast_path.py -q`
- `uv run pytest tests/meta/test_phase122_master_audit_guards.py tests/meta/test_governance_route_handoff_smoke.py tests/meta/test_governance_guards.py -q`

## Risks

- 若 guard 只检查 URL 后缀，不检查 tag 语义，下一次 release bump 仍可能回退到 `main`
- 若把 `hacs.json` 拉进 provenance 叙事过深，会模糊 metadata truth 的正式边界
- 若新的 focused guard 与现有通用 tests 职责重叠过多，后续维护成本会升高

## Done When

- 所有 release-facing metadata URL 都能回溯到当前 package semver 对应的 Git tag
- `hacs.json` 的角色被明确限制在 minimum-version truth，不再承担额外 provenance 故事
- focused guards 能在 ledger 缺项、public/appendix 边界回流、route drift 或 metadata traceability 漂移时直接失败
