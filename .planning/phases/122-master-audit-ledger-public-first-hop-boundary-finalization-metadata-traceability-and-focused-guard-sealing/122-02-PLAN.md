# Plan 122-02: 清理 public first-hop 边界并降噪 maintainer appendix

## Scope

- 固定对外 docs-first 路由：`README.md` / `README_zh.md` → `docs/README.md` → `CONTRIBUTING.md` / `docs/TROUBLESHOOTING.md` / `SUPPORT.md` / `SECURITY.md`
- 让 maintainer appendix 保持可达但只作为附录深链，不再与 contributor / user first-hop 并列竞争
- 保持 private-access honesty、continuity truth 与 release custody 真实可见，但不回流首屏叙事

## Files

- `README.md`
- `README_zh.md`
- `docs/README.md`
- `CONTRIBUTING.md`
- `SUPPORT.md`
- `SECURITY.md`
- `docs/TROUBLESHOOTING.md`
- `docs/MAINTAINER_RELEASE_RUNBOOK.md`
- `.planning/baseline/GOVERNANCE_REGISTRY.json`

## Implementation Steps

1. 统一 public first-hop 入口顺序与措辞：根 README 负责概览与跳转，`docs/README.md` 负责 canonical docs map，`CONTRIBUTING.md`/`docs/TROUBLESHOOTING.md`/`SUPPORT.md`/`SECURITY.md` 负责后续分流。
2. 从 `README.md`、`README_zh.md`、`docs/README.md`、`CONTRIBUTING.md` 的首屏剥离 maintainer-only continuity / custody / archived-evidence 细节，仅保留必要且不抢 first-hop 的显式 deep link。
3. 让 `SUPPORT.md` 与 `SECURITY.md` 的 public intake、response expectations、access-mode honesty 全部出现在 `## Maintainer Appendix` 之前；appendix 只保留 maintainer continuity / custody / freeze truth。
4. 若 docs-first / appendix 边界改变了 machine-readable 投影，同时同步 `.planning/baseline/GOVERNANCE_REGISTRY.json`，避免 registry、docs 与 meta tests 再次漂移。

## Validation

- `uv run pytest tests/meta/test_phase75_access_mode_honesty_guards.py tests/meta/toolchain_truth_docs_fast_path.py tests/meta/test_governance_release_docs.py -q`
- `uv run pytest tests/meta/test_governance_release_continuity.py tests/meta/test_governance_release_contract.py tests/meta/test_version_sync.py -q`

## Risks

- 双语入口若未同步，会出现 `README.md` 与 `README_zh.md` first-hop 叙事分叉
- appendix 若压得过深，可能让 maintainer continuity / custody truth 变得不可发现
- 若只改 prose、不改 registry-backed projection，现有 docs-fast-path tests 仍会继续漂移

## Done When

- public first-hop 在根 README 与 `docs/README.md` 首屏即清晰可见，且双语导航等价
- `SUPPORT.md` / `SECURITY.md` 的公开入口全部早于 `## Maintainer Appendix`
- maintainer appendix 仍可显式深链到 `docs/MAINTAINER_RELEASE_RUNBOOK.md` 与治理真源，但不再抢占 public route
