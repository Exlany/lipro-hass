<task type="auto">
  <name>Task 1: tighten diagnostics and system-health adapter signatures around existing control surfaces</name>
  <files>custom_components/lipro/diagnostics.py, custom_components/lipro/system_health.py, tests/core/test_diagnostics.py, tests/core/test_system_health.py</files>
  <action>把 `diagnostics.py` 与 `system_health.py` 中 broad `Any` / loose callable story 收紧为显式 typed adapter contract：优先复用 control-surface 现有返回形态或局部 type alias，保留 JSON-like diagnostics payload 的必要宽度，但不要让 helper/entrypoint 参数继续无约束。必须保持 `system_health.async_register()` 仍注册根模块自己的 `system_health_info`，不得直接把 control surface 函数对象塞进注册流程，也不得把两个根模块长回第二业务根。</action>
  <acceptance_criteria>
    - diagnostics and system-health entrypoints no longer rely on unconstrained Any where a local or imported typed contract is available
    - system-health callback identity semantics remain unchanged
    - focused diagnostics/system-health tests stay green
  </acceptance_criteria>
  <verify>`uv run pytest -q tests/core/test_diagnostics.py tests/core/test_system_health.py && uv run mypy --follow-imports=silent custom_components/lipro/diagnostics.py custom_components/lipro/system_health.py`</verify>
  <done>Thin control adapters are more typed and still outwardly identical.</done>
</task>
<task type="auto">
  <name>Task 2: formalize options-flow persisted option typing and schema builders</name>
  <files>custom_components/lipro/flow/options_flow.py, tests/flows/test_options_flow.py, tests/flows/test_options_flow_utils.py</files>
  <action>为 `LiproOptionsFlow` 的 `_options`、schema builder helpers、device-filter normalization 与 `async_create_entry()` payload 建立显式 scalar/text-selector option type；把 `dict[str, Any]` 限缩到 bounded options model 或局部 alias，不改变 `init` / `advanced` step UX、大小写归一、list truncation、selector 文本处理与现有 config-entry merge semantics。不要借 typing 改动 runtime/control ownership，也不要扩展新选项。</action>
  <acceptance_criteria>
    - options-flow main path no longer depends on broad Any for persisted option values
    - advanced and basic schemas keep the current behavior and normalization rules
    - focused flow tests and mypy confirm the narrowed contract
  </acceptance_criteria>
  <verify>`uv run pytest -q tests/flows/test_options_flow.py tests/flows/test_options_flow_utils.py && uv run mypy --follow-imports=silent custom_components/lipro/flow/options_flow.py`</verify>
  <done>Options flow keeps the same UX while using a more honest typed options contract.</done>
</task>
