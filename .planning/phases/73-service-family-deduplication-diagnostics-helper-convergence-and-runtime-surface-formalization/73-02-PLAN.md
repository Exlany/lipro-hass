# Plan 73-02: 收口 diagnostics/helper duplication，并压平 runtime-access projection surface

<objective>
在不新增第二 diagnostics/runtime facade root 的前提下，把 diagnostics、telemetry、system-health 与 runtime-access 之间重复 helper/support 编排压平，让 control readers 直接消费更诚实的 typed projection API，而不把 platform/entity 重新拉回 control locator。

Purpose: 按 `ARC-19` / `HOT-33` / `TYP-21` 继续降低 diagnostics/helper 认知噪音，同时保住 `runtime_access.py` 作为唯一 outward runtime read-model home。
Output: 更直白的 runtime-access projection helpers、更薄的 diagnostics/system-health adapters，以及 focused regression proof。
</objective>
