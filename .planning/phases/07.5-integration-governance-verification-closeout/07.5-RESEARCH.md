# Phase 7.5 Research

**Phase:** `7.5 Integration / Governance / Verification 收尾`
**Source milestone research:** `.planning/research/SUMMARY.md`
**Date:** 2026-03-13

## Findings

### 1. `07.5` 的职责是仲裁，不是再造实现

`07.5` 的核心不是“再加一个功能层”，而是把 `07.1-07.4` 已引入的 boundary / enforcement / telemetry / replay 资产纳入同一套治理与验收链：

`NORTH_STAR_TARGET_ARCHITECTURE -> baseline docs -> review ledgers -> automated guards/tests -> phase closeout evidence`

因此：
- `07.5` 必须优先回写 `AUTHORITY_MATRIX / PUBLIC_SURFACES / VERIFICATION_MATRIX / FILE_MATRIX / RESIDUAL_LEDGER / KILL_LIST`；
- 必须生成机器友好的 evidence index，列出 owner、authority、verification commands、residual gates；
- 不应越权重新定义 production behavior 或新建 runtime root。

### 2. v1.1 新资产最容易发生“三套口径分裂”

`07.1-07.4` 新增资产分布在多个层面：
- `core/protocol/boundary/*`
- architecture policy / meta guards / CI gate
- `core/telemetry/*` 与 control telemetry surface
- `tests/harness/protocol/*` 与 replay manifests

如果没有统一 closeout，最常见的退化是：
- 文档登记了 A，但 tests 仍靠 B；
- verification matrix 没有对应 replay / exporter evidence；
- residual/delete gate 未显式落表；
- AI 可分析的 evidence index 缺位，导致 `08` 只能重新扫描全仓。

### 3. 最优 closeout 产物应包括“机器友好 evidence index”

为对接 `08 AI Debug Evidence Pack`，`07.5` 最好显式产出：
- 一份 `v1.1 evidence index`，列出：
  - boundary authority sources
  - telemetry exporter views
  - replay run summaries
  - verification commands / test suites
  - residual items / delete gates
- 一份 phase closeout summary，说明哪些 truth 已锁定、哪些仍是后续演进点。

该 evidence index 必须是可 pull 的稳定入口，而不是散落在多份文档里等人手工拼装。

### 4. 最优实现仍是“轻治理”，不是再造治理框架

`07.5` 不需要引入新治理系统；最佳做法是：
- 继续使用现有 baseline/review docs；
- 强化 meta guards / verification matrix / file matrix 关联；
- 用一份增量 evidence index 把 v1.1 资产串起来；
- 让 `STATE / ROADMAP / REQUIREMENTS` 明确反映 phase 收尾状态与下一步命令。

### 5. 与 `08` 的边界必须先锁定

`07.5` 应拥有：
- 所有真源矩阵的一致性仲裁；
- v1.1 closeout evidence index；
- residual/delete gate 仲裁。

`08` 才拥有：
- AI debug evidence pack schema；
- exporter/collector 命令；
- 伪匿名 evidence pack 具体产物。

换言之：
- `07.5` 输出“给 AI / 工具消费的正式指针与索引”；
- `08` 输出“实际 evidence pack 打包器与包格式”。

## Risks To Avoid

- 用文档覆盖实现偏差，而不是用验证证据裁决；
- 把 `07.3 / 07.4` 的权责重新揉回 `07.5`，造成 phase ownership 模糊；
- 只更新一半矩阵，导致 owner / authority / verification 仍然分裂；
- 为了 AI evidence 需求，提前在 `07.5` 自己实现一套 evidence pack exporter。

## Validation Architecture

`07.5` 的验证必须同时证明：
1. `07.1-07.4` 新资产都已登记 owner / authority / verification / residual；
2. meta guards 能阻断治理真源漂移；
3. `v1.1 evidence index` 能稳定列出 closeout 所需的上游证据与命令；
4. `STATE / ROADMAP / REQUIREMENTS` 与治理矩阵保持一致。

## Final Arbitration

`07.5` 的最优实现不是“写更多说明文档”，而是：
- 把 `07.1-07.4` 的新资产完整纳入治理真源；
- 产出可机器消费的 v1.1 evidence index；
- 明确 residual / delete gate / acceptance evidence，给 `08` 提供稳定、无歧义的上游索引。
