# Phase 7.1 Context

**Phase:** `7.1 Protocol Boundary Schema/Decoder 收口`
**Milestone:** `v1.1 Protocol Fidelity & Operability`
**Date:** 2026-03-13

## User / Product Context

- 当前 v1.0 已完成北极星重建与治理 closeout；下一步不是继续清旧债，而是提升协议保真、观测与可回放能力。
- 用户明确要求：新能力仍必须服从最先进但不失边界诚实的架构，不允许重新长出多套合法实现。

## Architectural Context

- `LiproProtocolFacade` 仍是唯一正式 protocol root
- `Coordinator` 仍是唯一正式 runtime root
- decoder family 必须是 protocol boundary collaborator，不是新 façade/root
- raw vendor payload 不得穿透到 runtime/domain/entity/control

## Phase Goal

把 REST/MQTT 的 decode authority 收口到 `core/protocol/boundary/`，形成 schema/decoder family、registry 与 decode result，同时为后续 telemetry export 与 replay harness 提供统一观察点。

## Non-Negotiable Constraints

- 不得破坏既有 canonical contracts
- 不得让 decoder 私有类型泄漏到 runtime/domain/entity
- 不得新增第二协议主链或旁路 normalizer
- 新增资产必须同步 authority / verification / file-matrix / residual docs
