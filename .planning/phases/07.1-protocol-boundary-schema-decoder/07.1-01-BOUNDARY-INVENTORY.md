# 07.1-01 Boundary Inventory

## Scope

- 本清单只盘点 `REST/MQTT` 的 decode / normalize authority，不改变 `LiproProtocolFacade` 作为唯一正式 protocol root 的裁决。
- `core/protocol/boundary/` 是新的 formal home；`core/api/**`、`core/mqtt/**` 暂保留接线与 legacy decode 实现，后续由 `07.1-02` 逐步迁入 decoder family。
- runtime / domain / control 继续只消费 canonical contracts，不直接理解 decoder 私有类型、schema backend 或 raw vendor payload。

## Target Responsibility Split

| Component | Owns | Must not own |
|---|---|---|
| `BoundarySchemaRegistry` | family identity、versioning、authority source、drift-flag taxonomy 的登记 | transport IO、HA 生命周期、runtime/domain/control 解释逻辑 |
| `RestBoundaryDecoder` / `MqttBoundaryDecoder` | raw input -> canonical contract 的 decode / validate / normalize | 成为第二 protocol root、直接驱动 telemetry sink、绕过 `LiproProtocolFacade` 对上暴露 |
| `BoundaryDecodeResult` | canonical output + stable decode evidence（family/version key、authority、drift flags、fingerprint、notes） | 作为 runtime/domain public type 泄漏到协议面之外 |
| `BoundaryDecoderKey` | decoder family / version 的稳定键空间 | 决定业务语义或替代 fixture authority |

## Migration Sequence

1. `07.1-01`：建立 inventory、schema registry / decoder / result contract、authority/file-matrix 落表。
2. `07.1-02`：把至少一条 REST family 与一条 MQTT family 接到 decoder family，并阻断 raw payload leakage。
3. `07.1-03`：补 boundary fixtures、contract assertions、replay-ready evidence，使 authority 从“当前测试覆盖”升级为“正式 decoder fixture family”。

## Handoff Notes

- `07.2` enforcement 直接消费 registry / inventory 中的 family 与 owner 信息，阻止 decode authority 再次散落。
- `07.3` telemetry exporter 应复用 `BoundaryDecodeResult` / `BoundaryDecoderDescriptor`，而不是重新发明第二套 decode 观察点。
- `07.4` replay harness 应复用 registry family id 与 version key，避免 REST/MQTT replay corpus 各自生长第二真源。
