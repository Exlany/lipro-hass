# Phase 7.1 Architecture

## Target Shape

- `custom_components/lipro/core/protocol/boundary/`
  - `schema_registry.py`
  - `rest_decoder.py`
  - `mqtt_decoder.py`
  - `result.py`

## Responsibility Split

- `transport / endpoints / mqtt message processors`：仅负责接线与传递原始输入
- `boundary decoders`：完成 decode / validate / normalize / fingerprint / drift flag
- `LiproProtocolFacade`：作为唯一 protocol root，向上暴露 canonical contract
- `runtime plane`：只消费 canonical contract，不理解 decoder backend

## Acceptance Shape

- decoder family 有统一 registry
- REST/MQTT 输入路径有正式接点
- decode result 可被 telemetry / replay 共用
- governance/authority 文档同步完成
