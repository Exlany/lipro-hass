# Plan 89-01: formalize entity-facing runtime verbs and close the OTA query seam

<objective>
把 entity-facing runtime surface 收口成诚实的 public verbs：实体只知道“发送命令 / 应用乐观状态 / 查询 OTA”，不再知道 runtime service 对象、device lock 或 protocol seam。

Purpose: 落实 `ARC-23`，堵住 entity/runtime/protocol boundary leakage，同时保住现有 optimistic update、debounce 与 OTA row selection 语义。
Output: 更窄的 `runtime_types` 协议、迁移后的 entity/update 实现，以及阻止回流的 focused guards。
</objective>
