# Plan 04-01: 建立 CapabilityRegistry / CapabilitySnapshot，并把 device slice 接到单一真源

## Preconditions

已满足：
- `Phase 1 / 1.5` baseline asset pack 已完成；
- `Phase 2 / 2.5 / 2.6` protocol 与 external-boundary truth 已完成；
- `Phase 3` control-plane closeout 已完成；
- `ROADMAP / REQUIREMENTS / PUBLIC_SURFACES / TARGET_TOPOLOGY` 已把 `CapabilityRegistry / CapabilitySnapshot` 声明为终态目标。

## Tasks

1. **建立 formal capability home**
   - 新增 `custom_components/lipro/core/capability/`；
   - 落地 `CapabilitySnapshot` 与 `CapabilityRegistry`；
   - 明确 category/platforms/capability flags 的 canonical owner。

2. **降级 legacy device capability surface**
   - 把 `custom_components/lipro/core/device/capabilities.py` 改成 compat bridge；
   - 保留历史导入名，但不再保留第二套实现逻辑。

3. **接通 device slice**
   - `device_snapshots.py` 通过 registry 构造 capability snapshot；
   - `device_views.py` 的 `category()` / `platforms()` / `fan_speed_range()` 走 capability truth；
   - `device.pyi` 对齐正式 capability surface。

4. **补最小充分测试**
   - 新增 `tests/core/capability/test_registry.py`；
   - 扩展 `tests/core/device/test_capabilities.py`；
   - 验证 capability truth、compat bridge、device views 三者对齐。

5. **同步治理与 baseline**
   - 标记 `04-01` 完成；
   - 回写 `STATE / ROADMAP / TARGET_TOPOLOGY / PUBLIC_SURFACES / VERIFICATION_MATRIX / FILE_MATRIX / RESIDUAL_LEDGER`。

- `uv run pytest tests/core/device/test_state.py tests/platforms/test_sensor.py tests/platforms/test_entity_behavior.py tests/platforms/test_platform_entities_behavior.py -q`

## Success Criteria

- `core/capability/` 已成为 capability truth 的正式实现 home；
- device slice 不再各自重算 category/platforms；
- compat bridge 被明确限制在 legacy import surface；
- Phase 4 的执行已从“只有路线图目标”升级为“有真实代码与测试支撑的 formal start”。
