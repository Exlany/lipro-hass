# Plan 04-02: 迁移 entity / platform / helper 到统一 capability projection

## Goal

把平台、实体、descriptor、helper 中直接散落的 capability 规则迁移到 `CapabilitySnapshot` 或围绕其构建的 projection，终结“同一能力语义在多个层重复长出来”。

## Scope

- `custom_components/lipro/entities/**/*.py`
- `custom_components/lipro/{light,fan,binary_sensor,cover,climate,select}.py`
- `custom_components/lipro/helpers/platform.py`
- 必要的 domain projection helpers / command contracts
- 对应 platform/entity tests

## Must Haves

- 平台/实体不再自行解释 category/platforms/capability flags；
- descriptor / projection helpers 围绕正式 snapshot 建模；
- 所有新增能力入口集中到 capability model，而不是散落到多个平台分支。

## Expected Verification

- `uv run pytest tests/platforms/test_fan.py tests/platforms/test_sensor.py tests/platforms/test_entity_behavior.py tests/platforms/test_platform_entities_behavior.py -q`
- `uv run pytest tests/core/capability tests/core/device tests/core/test_device.py -q`
