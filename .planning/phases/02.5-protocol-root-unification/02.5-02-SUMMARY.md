# Phase 02.5 Plan 02.5-02 执行总结

## 本次完成

- 新建 `custom_components/lipro/core/protocol/`，落地 `__init__.py`、`facade.py`、`session.py`、`telemetry.py`、`diagnostics_context.py`、`contracts.py`、`compat.py`，把 unified root 从 architecture prose 变成真实代码。
- 新增 `LiproProtocolFacade`：统一持有 `ProtocolSessionState`、`RequestPolicy`、`ProtocolTelemetry`、`ProtocolDiagnosticsContext` 与 `CanonicalProtocolContracts`，并把 `LiproRestFacade` 降回 REST child façade。
- 新增 `LiproMqttFacade`：把 concrete `LiproMqttClient` 包装成 protocol-owned child façade，使 MQTT 生命周期、telemetry 与 diagnostics context 都在 root-owned truth 下协作。
- 改写 `custom_components/lipro/core/api/client.py`，允许注入 `ClientSessionState` 与 `RequestPolicy`，让 shared auth / pacing truth 由 protocol root 主动拥有，而不是继续由 REST child 自建。
- 改写 `custom_components/lipro/core/coordinator/mqtt_lifecycle.py`，由 `client.build_mqtt_facade(...)` 构造 MQTT child façade，并在 setup 失败时回收 protocol root 上的 MQTT attach state。
- 在 `tests/core/api/test_protocol_contract_matrix.py` 中新增 unified-root proof，明确 root-owned session/request-policy 与 canonical MQTT contract normalization 已落地。

## 关键裁决

- **shared truth 裁决**：session / request policy 的 owner 已上收到 `LiproProtocolFacade`，`LiproRestFacade` 只消费注入的 shared truth，不再自定协议级 owner。
- **MQTT owner 裁决**：`LiproMqttClient` 不再是 runtime-facing formal root；它只是 `LiproMqttFacade` 内部 concrete transport。
- **diagnostics/telemetry 裁决**：protocol-level telemetry 与 diagnostics context 已有 root-owned 载体，避免继续沿 REST / MQTT 各自生长。
- **canonical contract 裁决**：canonical MQTT config normalization 与 schedule/helper normalization 已集中到 `CanonicalProtocolContracts`，作为统一协议根的正式 contract helper。
