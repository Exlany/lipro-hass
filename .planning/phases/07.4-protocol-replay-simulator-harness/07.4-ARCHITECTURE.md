# Phase 7.4 Architecture

## Architectural Goal

把“逆向协议样本”正式升级为 assurance-plane replay harness：通过 authority-indexed corpus 与 deterministic driver，走同一条生产 public path，输出 canonical / drift / telemetry 三类可仲裁证据。

## Single Truth Chain

```text
AUTHORITY_MATRIX + boundary inventory
  -> authority fixtures
     - tests/fixtures/api_contracts/*
     - tests/fixtures/protocol_boundary/*
  -> replay manifests
     - tests/fixtures/protocol_replay/*
  -> deterministic driver
     - tests/harness/protocol/replay_loader.py
     - tests/harness/protocol/replay_driver.py
  -> formal production path
     - LiproProtocolFacade
     - core/protocol/boundary/*
  -> assertions
     - canonical contract assertions
     - drift assertions
     - telemetry assertions (via 07.3 exporter)
  -> outputs
     - replay run summary
     - integration regression evidence
```

裁决：
- authority fixtures 仍是正式输入真源；
- replay manifests 只做场景编排与版本/seed/clock 元数据，不复制 authority payload；
- deterministic driver 只做 assurance orchestration；
- telemetry assertions 必须消费 `07.3` exporter truth；
- replay run summary 是 `07.5 / 08` 的正式上游证据之一。

## Formal Components

### 1. `tests/fixtures/protocol_replay/*`

正式职责：
- 为 replay scenario 提供 manifest/index；
- 记录 authority pointer、scenario id、version、seed、clock baseline、expected assertion families；
- 不复制 `api_contracts` / `protocol_boundary` 已拥有的 payload truth。

### 2. `tests/harness/protocol/replay_loader.py`

正式职责：
- 解析 replay manifests；
- 校验 authority pointers 是否存在且合法；
- 把场景装配成 driver 可执行输入。

### 3. `tests/harness/protocol/replay_driver.py`

正式职责：
- 在固定 seed / time baseline / ordering 下执行 replay；
- 只调用正式 façade / decoder public path；
- 产出 canonical / drift / telemetry assertions 所需上下文。

### 4. `tests/harness/protocol/replay_assertions.py`

正式职责：
- 统一 contract assertions、drift assertions 与 telemetry assertions；
- 避免 REST/MQTT tests 各自维护一套 ad-hoc 断言逻辑；
- 对 telemetry 仅消费 exporter / control telemetry surface，不直接摸 runtime internals。

### 5. `tests/harness/protocol/replay_report.py`

正式职责：
- 生成 replay run summary；
- 输出 scenario、family、version、result、drift flags、关键 telemetry 对齐结果；
- 成为 `07.5` evidence index 与 `08` evidence pack 的上游输入。

## Cross-Phase Arbitration

### `07.3` owns telemetry truth

`07.4` 只能：
- 读取 exporter snapshot / views；
- 断言 exporter truth 是否与 replay scenario 对齐；
- 不改写 exporter field semantics、redaction policy、cardinality budget。

### `07.4` owns replay truth

`07.4` 必须拥有：
- replay manifests；
- deterministic driver；
- replay run summary schema；
- replay-specific authority/drift assertions。

### `07.5` owns governance closeout

`07.4` 只做最小必要 governance hook：
- authority / verification / file ownership 的最小登记；
- 未完成的 residual / delete gate 留给 `07.5` 统一仲裁。

### `08` owns AI Debug packing

`07.4` 只需保证 replay run summary 结构稳定、可 pull、可追溯；
不直接承担 evidence pack schema 或打包命令。

## Deterministic Rules

- 同一 scenario 必须固定 seed、时间基准与输入顺序；
- 比较逻辑必须使用 canonical output，而不是原始 vendor payload；
- telemetry assertions 允许真实时间戳，但仍必须遵守 `07.3` 的敏感字段与伪匿名引用规则；
- replay manifests 的版本戳必须是 authority / drift 审核的一部分。

## Non-Goals

- 不引入 broker/cloud 全量仿真平台；
- 不把 replay harness 放进 `custom_components/lipro/` 生产主链；
- 不把 replay 变成新的 diagnostics implementation 或第二 protocol root。
