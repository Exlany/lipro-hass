# Summary 07.4-01

## Outcome

- 新增 `tests/harness/protocol/` 作为 assurance-only replay harness formal home：`replay_models.py`、`replay_loader.py` 与 `replay_driver.py` 已把 replay 场景的“装载 / 排序 / 执行”职责收口到单一测试家园。
- 新增 `tests/fixtures/protocol_replay/` manifest family，并落地 3 个代表性 scenario：REST `get_mqtt_config` 的 direct / wrapped 回放，以及 MQTT `device_state` 边界回放；manifests 只记录 authority pointer、family、version、seed、clock baseline 与 assertion families，不复制第二份 payload truth。
- `ProtocolReplayDriver` 已明确 deterministic controls：固定 scenario metadata、固定 clock baseline，并且只走 `LiproProtocolFacade.contracts.normalize_mqtt_config` 与 `core.protocol.boundary.decode_mqtt_properties_payload` 这两条正式 public path。
- `.planning/baseline/AUTHORITY_MATRIX.md` 已新增 replay manifest family 的 authority 关系；`scripts/check_file_matrix.py` 已为 `tests/harness/protocol/*` 与后续 `07.4` replay tests 预登记 `Phase 7.4` owner。
