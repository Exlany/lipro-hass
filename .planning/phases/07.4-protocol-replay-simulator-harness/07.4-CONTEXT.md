# Phase 7.4 Context

**Phase:** `7.4 Protocol Replay / Simulator Harness 建立`
**Milestone:** `v1.1 Protocol Fidelity & Operability`
**Date:** 2026-03-13
**Status:** Ready for planning

## Product / User Context

本 phase 的目标是把“逆向协议样本”升级为 **可确定性回放** 的 assurance 资产：

- 维护者可以离线重放 REST/MQTT flows，复现协议漂移与运行时行为；
- CI 可以用 replay corpus 做 regression evidence；
- 回放必须复用正式生产主链（`protocol facade -> boundary decoder -> canonical contracts`），禁止复制第二套实现。

## North Star 2.0 + AI Debug Decisions

契约者裁决本仓库只面向 Home Assistant；因此 replay harness 的价值不在“跨平台 SDK”，而在“HA 集成可观测、可仲裁、可回放”。

- **AI 分析友好**：replay 产物必须能生成结构化 evidence（包含时间戳、序列、错误分类、关键指标），供后续 AI 辅助定位。
- **隐私与脱敏优先**：replay corpus 不得包含 token/secret/`password_hash` 等凭证等价物；允许使用伪匿名化引用（报告内稳定，跨报告不可关联）。
- **与 7.3 exporter 对齐**：replay 的 telemetry assertions 必须复用 `RuntimeTelemetryExporter` 输出，不得再造第二套 telemetry truth。

## Phase Goal

1. 建立 replay corpus 的 authority source / 版本戳 / deterministic controls；
2. 通过正式 façade/decoder 路径重放 representative REST/MQTT flows；
3. 产出 fidelity 与 operability evidence（含 telemetry assertions）。

## Non-Negotiable Constraints

- replay harness 只属于 assurance plane，不得获得 runtime 编排权；
- 不得通过 replay 重新引入 raw payload leakage 或 compat shell truth；
- 不得为了“更先进”而引入与 HA 插件不匹配的重型仿真栈。
