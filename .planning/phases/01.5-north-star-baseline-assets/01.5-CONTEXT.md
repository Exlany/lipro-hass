## Phase Boundary

本阶段不是“顺手写几份文档”，而是：

**把北极星终态的拓扑、依赖方向、正式 public surfaces、验证闭环与 authority 同步规则，收敛为后续所有 phase 都必须引用的正式基准资产包。**

本阶段必须完成：
- 将 `TARGET_TOPOLOGY / DEPENDENCY_MATRIX / PUBLIC_SURFACES / VERIFICATION_MATRIX / AUTHORITY_MATRIX` 固化为正式 baseline asset pack
- 把 `BASE-01 / BASE-02 / BASE-03 / ARCH-04 / ASSR-06` 变成可追踪 phase 交付，而不是散落参考文档
- 为 dependency direction 与 public surfaces 建立最小可运行 seed guards，减少后续 phase 全靠人工审查
- 为 `Phase 2 / 2.5 / 2.6 / 3` 提供可直接引用的 baseline truth

本阶段明确**不做**：
- 直接重构 protocol / runtime / control / domain 实现代码
- 全量 architecture enforcement / CI gate 正式化（属于 Phase 6）
- file-level `378/378` 治理矩阵清零（属于 Phase 7）
- 改写协议 contract baseline（属于 Phase 1）
</domain>
