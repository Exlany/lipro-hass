# Phase 01.5 Architecture: Baseline Asset Pack and Seed Guards

## Objective

把北极星基准从“已有几份 baseline markdown”升级为“后续 phase 必须引用、可被最小 guards 监督、具备明确 handoff 的正式资产包”。

## Requirement Alignment

| Requirement | What This Phase Must Decide | Expected Output |
|---|---|---|
| `BASE-01` | 五平面目标组件、formal roots、目录映射、canonical surfaces 是否已单口径 | topology + public surfaces baseline |
| `BASE-02` | allowed / forbidden dependency truth 是否显式、可被 guard 引用 | dependency matrix + seed guards |
| `BASE-03` | verification matrix 是否成为 phase 验收真源 | verification matrix + handoff mapping |
| `ARCH-04` | 重构前是否已有 baseline-level architectural guard | dependency/public-surface seeds |
| `ASSR-06` | 最小可运行 dependency/public-surface guards 是否存在 | tests/meta seed suite |

## Baseline Asset Pack

```text
baseline asset pack
├── TARGET_TOPOLOGY.md
├── DEPENDENCY_MATRIX.md
├── PUBLIC_SURFACES.md
├── VERIFICATION_MATRIX.md
└── AUTHORITY_MATRIX.md
```

## Baseline Asset Role Contract

Phase 1.5 不在 phase docs 内再创造第二套真相，而是把北极星原则翻译成正式 baseline inputs：

- `docs/NORTH_STAR_TARGET_ARCHITECTURE.md` 裁决 north-star principles。
- `.planning/baseline/*.md` 是 downstream phases 必须引用的单口径真源。
- phase docs / summaries 只负责解释、验收与回写，不得平行改写 root / public-surface / authority 语义。

## Formal Ownership Rules

| Asset | Formal Role | Must Not Become |
|---|---|---|
| `TARGET_TOPOLOGY.md` | 五平面与目录意图真源 | 任意单 phase 的临时草图 |
| `DEPENDENCY_MATRIX.md` | 允许/禁止依赖真源 | 仅 reviewer 心智模型 |
| `PUBLIC_SURFACES.md` | canonical / transitional / forbidden surfaces 真源 | 被 compat shell 反向改写 |
| `VERIFICATION_MATRIX.md` | requirement → artifact → test → phase 验收真源 | 过期 checklist |
| `AUTHORITY_MATRIX.md` | docs/fixtures/generated/implementation 同步真源 | 只在审计时才查看的边缘文档 |

## Guard Design

### 1. Dependency Seeds
在 `tests/meta/` 中落最小 guards，先阻止最明显的 forbidden directions：
- entity/platform 直连 protocol internals
- control plane 直连 protocol internals
- compat shell 重新成为 canonical surface

### 2. Public Surface Seeds
最小检查应能回答：
- canonical surfaces 是否仍与 `PUBLIC_SURFACES.md` 一致
- forbidden roots 是否被测试或文档重新提升
- baseline docs 与 phase docs 是否出现明显双口径

### 3. Verification as Handoff
`VERIFICATION_MATRIX.md` 不只描述 phase，还要回答：
- 当前 phase 完成后，下游 phase 应引用什么
- 无变更时如何说明“为何无变化”

## Execution Slices

### Plan 01.5-01
- 收束 baseline asset pack 的内容、状态、同步方向
- 让 topology / public surfaces / authority 成为正式资产

### Plan 01.5-02
- 建立 dependency / public-surface seed guards
- 让 baseline 不再完全依赖人工审查

### Plan 01.5-03
- 建立 verification matrix 与 phase acceptance/handoff 映射
- 为 Phase 2+ 提供正式 baseline closeout proof

## Downstream Handoff

### Handoff to Phase 02
Phase 02 必须引用：
- `TARGET_TOPOLOGY.md`
- `DEPENDENCY_MATRIX.md`
- `PUBLIC_SURFACES.md`
- `VERIFICATION_MATRIX.md`
- `AUTHORITY_MATRIX.md`

### Handoff to Later Phases
`2.5 / 2.6 / 3` 不再各自定义 public-surface 或 dependency 真相，只能在 baseline docs 上扩展并回写。

## Exit Conditions

只有以下条件同时满足，Phase 1.5 才算 closeout：
- 五份 baseline docs 都已进入正式 phase 验收语义
- `tests/meta/` 至少具备 dependency/public-surface 最小 guards
- `VERIFICATION_MATRIX.md` 已能驱动后续 phase 验收
- downstream phases 已获得具体可引用的 baseline handoff artifacts
