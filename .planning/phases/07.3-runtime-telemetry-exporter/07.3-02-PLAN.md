<objective>
把 `auth recovery`、`MQTT reconnect`、`command confirmation latency`、`refresh latency` 等真实运行信号正式接入 exporter，并把 diagnostics/system health/developer report/anonymous-share 的多口径 truth 收敛成“同一 exporter 投影”；同时补齐 redaction/cardinality 校验与黑盒主链集成验证，确保证据不是白盒 patch 堆出来的。
</objective>

<tasks>
<task>把 runtime/protocol 四类核心信号对齐为 exporter-ready 结构化 source：
  1) `auth recovery`：在 `core/api/client_auth_recovery.py` 提供结构化 outcome（成功/失败/原因分类/重试次数/耗时区间），供 exporter 消费。
  2) `command confirmation latency`：把 `core/command/confirmation_tracker.py` 中的确认耗时/超时原因纳入 runtime telemetry source（不要只停留在 command traces）。
  3) `MQTT reconnect`：从 `mqtt_runtime.py` 的 reconnect-ish state 提取 exporter 合约字段（connected/backoff/last error type/last disconnect time），避免 raw exception 或 id 泄露。
  4) `refresh latency`：从 `runtime/tuning/metrics.py` 与 coordinator refresh 计时点收口为稳定指标字段，并明确窗口/单位/采样策略。</task>
<task>改造 consumers 以消除“三套 telemetry 口径并存”风险：
  - `control/diagnostics_surface.py`、`control/system_health_surface.py`：只消费 exporter views，不再拼 telemetry truth。
  - `diagnostics.py`、`system_health.py`：继续保持 thin adapter，调用 control surfaces 输出。
  - `core/utils/developer_report.py`：转为 formatter / sink adapter，不再自带另一套事实来源。</task>
<task>升级 redaction/cardinality 策略并补齐 tests：
  - 强化 `control/redaction.py` 对 `password_hash`（凭证等价物）的处理：不仅 redact，还要阻止进入 exporter view 的高-cardinality 字段。
  - 为 exporter sinks 增加 targeted regression tests：确保 phone/device id/token/raw error payload 不进入 views。</task>
<task>新增黑盒主链集成测试：`tests/integration/test_telemetry_exporter_integration.py` 至少覆盖：
  - `setup_entry -> runtime_data -> exporter -> diagnostics/system_health` 路径成立；
  - 断言 diagnostics/system-health 的 telemetry payload 来自同一 exporter view；
  - 明确不依赖直接读取 coordinator 私有字段才能通过。</task>
<task>回写治理证据：更新 `.planning/baseline/VERIFICATION_MATRIX.md` 把 `OBS-01/OBS-02` 的 evidence 显式映射到 exporter unit + control + integration tests；更新 `.planning/reviews/RESIDUAL_LEDGER.md` 关闭“telemetry truth 分裂”残留项，或为无法在本 phase 完成的 consumer 留下 delete gate（供 07.5 closeout）。</task>
</tasks>

## Verification Criteria

- exporter 覆盖 roadmap 要求四类核心信号：`auth recovery`、`MQTT reconnect`、`command confirmation latency`、`refresh latency`；
- diagnostics / system health / developer report 等 consumers 共用 exporter truth，不再各自定义事实；
- `password_hash` 等凭证等价字段不会出现在 exporter sink payload 或高-cardinality 字段中；
- 至少一条黑盒集成测试证明从 HA 入口到 exporter sinks 的路径成立，且不依赖 runtime internals backdoor；
- 本次改动不会把逻辑继续塞回已知热点（`client.py`/`coordinator.py`/`services/wiring.py`）而造成二次膨胀。

## Must Haves

- four core signals exported
- one truth across sinks
- redaction + cardinality tests
- black-box mainline evidence
- no legacy hotspot re-growth
