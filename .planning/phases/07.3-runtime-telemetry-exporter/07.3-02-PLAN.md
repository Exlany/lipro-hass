# Plan 07.3-02: 接上真实运行信号，收口所有消费口径，并为 07.4/07.5 铺好证据链

## Preconditions

已满足：
- `07.3-01` 已落地 exporter contracts / ports / sinks / governance baseline；
- protocol/runtime telemetry seeds 已存在于 `core/protocol/telemetry.py`、`diagnostics_context.py`、`coordinator/services/telemetry_service.py` 等正式协作者中；
- `07.3-CONTEXT.md` 已明确 exporter 只拉取快照与有界摘要，不得成为第二真相源。

## Tasks

1. **把四类核心信号接成 exporter-ready sources**
   - 在 `core/api/auth_recovery.py` 产出结构化 `auth recovery` 结果（成功/失败分类、重试次数、耗时区间）；
   - 在 `core/command/confirmation_tracker.py` 补齐 `command confirmation latency` 与超时原因；
   - 从 `core/coordinator/runtime/**` 与 `core/protocol/*.py` 收口 `MQTT reconnect` 与 `refresh latency` 的稳定字段、单位、窗口与采样策略；
   - 保证 source 只暴露快照/摘要，不输出 raw exception、raw payload、真实设备标识或凭证等价物。

2. **把 consumers 全部收口到同一 exporter truth**
   - 改造 `control/runtime_access.py`、`control/diagnostics_surface.py`、`control/system_health_surface.py` 使其只消费 exporter views；
   - 保持 `diagnostics.py`、`system_health.py` 为 thin adapter；
   - 将 `core/utils/developer_report.py` 降为 formatter / sink adapter，禁止继续维护独立事实来源。

3. **强化 redaction / pseudonymization / AI-debug 规则并补齐回归**
   - 在 `control/redaction.py` 明确 `password_hash`、token、secret、refresh/access key 为绝对禁止字段；
   - 测试 `entry_ref` / `device_ref` 仅报告内稳定、跨报告不可关联，同时允许 sinks 保留真实时间戳用于 AI 调试与 replay 对齐；
   - 为 diagnostics/system-health/exporter sinks 增加 targeted regression，防止 phone、device id、raw error payload、高基数字段泄露。

4. **补齐黑盒主链证据并回写 closeout 指针**
   - 新增 `tests/integration/test_telemetry_exporter_integration.py`，覆盖 `setup_entry -> runtime_data -> exporter -> diagnostics/system_health`；
   - 断言 diagnostics / system health 使用的是同一 exporter view，而不是直接读取 coordinator 私有字段；
   - 回写 `.planning/baseline/VERIFICATION_MATRIX.md` 与 `.planning/reviews/RESIDUAL_LEDGER.md`，为 `07.4 replay harness` 与 `07.5 closeout` 留下可复用 evidence / delete gate 指针。

## Success Criteria

- exporter 已覆盖 roadmap 要求的四类核心信号，并且字段、单位、窗口稳定；
- diagnostics / system health / developer report 统一从 exporter 拉取真相，不再分叉；
- 真实时间戳可保留，但 `password_hash`、token、secret、raw id/raw payload 不会出现在任何 sink payload；
- 至少一条黑盒主链测试证明 HA 入口到 exporter sinks 的路径成立，且不依赖 internals backdoor；
- `07.4` 可直接复用 exporter 输出做 replay assertions，`07.5` 可据此完成 residual closeout。
