# Phase 7.3 Architecture

## Target Shape

- `custom_components/lipro/core/telemetry/`
  - `models.py`
  - `ports.py`
  - `exporter.py`
  - `sinks.py`
- `custom_components/lipro/core/protocol/telemetry.py`
  - 继续作为 protocol source
- `custom_components/lipro/core/coordinator/services/telemetry_service.py`
  - 继续作为 runtime source
- `custom_components/lipro/control/runtime_access.py`
  - 退化为 thin access adapter / exporter locator
- `custom_components/lipro/diagnostics.py`
  - 只导出 diagnostics sink projection
- `custom_components/lipro/system_health.py`
  - 只导出 system-health sink projection

## Responsibility Split

- `ProtocolTelemetry`：记录 transport/auth/protocol 侧 summary 与 event seeds
- `CoordinatorTelemetryService`：记录 runtime orchestration metrics、signals 与 command traces
- `RuntimeTelemetryExporter`：合并 protocol/runtime sources，生成稳定 snapshot / sink views / budgeted projections
- `TelemetrySink`：把 exporter 结果投影到 diagnostics、system health、developer、CI 等消费场景
- `control.redaction`：继续拥有 HA-facing redaction policy；exporter 只能消费显式 redaction contract，不能自造第二套脱敏真相

## North Star 2.0 Additions (AI Debug Ready)

- exporter 输出必须是 **AI 分析友好** 的结构化 schema（稳定字段、明确单位、可版本化）。
- sinks 允许输出 **真实时间戳**（用于定位时序与回放对齐），但必须遵守：
  - 不输出凭证等价物（`password_hash`、token、secret、refresh/access key 等）；
  - 不输出可长期关联的真实标识。
- 采用 **伪匿名化引用**：
  - `entry_ref` / `device_ref` 作为 sinks 的默认关联键；
  - `*_ref` 必须在“报告内稳定、跨报告不可关联”。
- exporter 以 **pull 为主**：从 sources 拉取快照与 ring-buffer 摘要；sources 才可接收 push 事件，但必须有上限与清理策略。

## Acceptance Shape

- diagnostics / system health / exporter 共用单一 telemetry truth
- exporter 不读取 compat shell，不获得 runtime 编排权
- `password_hash` 等高敏字段不会出现在 exporter sink payload 或高-cardinality labels
- telemetry 复杂度从 `client.py` / `coordinator.py` / `wiring.py` 外移到 `core/telemetry/*`
- `07.4` 可直接复用 exporter 输出做 replay telemetry assertions
