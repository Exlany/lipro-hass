<objective>
建立 `07.3` 的单一 telemetry exporter formal home：用显式 contracts/ports 把 protocol/runtime 已有 telemetry 种子收口为可复用的 exporter truth，并为 diagnostics/system-health/developer/CI sinks 提供同一投影视图；同时避免继续膨胀 `client.py`、`coordinator.py` 与 `services/wiring.py`。
</objective>

<tasks>
<task>新增 `custom_components/lipro/core/telemetry/`，以 `models.py/ports.py/exporter.py/sinks.py` 形成最小但正式的 exporter family：只允许“读取/投影/预算化”，不允许获得 runtime 编排权（observer-only surface）。</task>
<task>在 `models.py` 定义稳定 contracts：至少包含 `TelemetrySnapshot`（protocol/runtime 合并快照）、`TelemetryViews`（diagnostics/system-health/developer/CI 视图）、`TelemetrySensitivity`（把 `password_hash` 视为凭证等价物）、`CardinalityBudget`（限制 label/key 高基数扩散）与版本戳字段（为 07.4 replay telemetry assertions 预留）。</task>
<task>在 `ports.py` 定义显式 ports：`ProtocolTelemetrySource`（来自 `ProtocolTelemetry + ProtocolDiagnosticsContext`）、`RuntimeTelemetrySource`（来自 `CoordinatorTelemetryService`）与 `TelemetrySink`；明确 source 只能“暴露快照/事件摘要”，不能要求下游读取内部协作者或 raw payload。</task>
<task>在 `exporter.py` 建立 `RuntimeTelemetryExporter`：合并 sources、执行 sensitivity/cardinality 预算、产出稳定 snapshot + sink views；禁止把 exporter 变成“日志堆叠器/事件总线/DI 容器”。</task>
<task>新增 `control/telemetry_surface.py`：作为 control-plane 唯一 bridge，封装“如何从 config entry/runtime root 定位 exporter 并取 view”的逻辑；确保 `diagnostics_surface/system_health_surface` 后续只走该 surface，不再通过 `getattr` 漫游 runtime internals。</task>
<task>补齐 `tests/core/telemetry/` 的基础测试骨架：至少验证 contracts 的稳定字段集合、cardinality budget 生效、敏感字段在 views 中不会泄露（尤其 `password_hash`）。</task>
<task>同步治理真源：
  1) 更新 `.planning/baseline/DEPENDENCY_MATRIX.md`：把 `core/telemetry/*` 明确定性为 assurance/observer hook（允许依赖 protocol/runtime sources；禁止 runtime/protocol 反向依赖 exporter）。
  2) 更新 `.planning/baseline/PUBLIC_SURFACES.md`：登记 `RuntimeTelemetryExporter` 与 `TelemetrySurface` 的 canonical public role。
  3) 更新 `.planning/baseline/VERIFICATION_MATRIX.md`：把 `OBS-01/OBS-02` 的证据映射到 exporter tests + control/integration tests。
  4) 更新 `.planning/reviews/FILE_MATRIX.md`：登记新文件 owner/plane。
  5) 更新 `.planning/reviews/RESIDUAL_LEDGER.md`：登记“多口径 telemetry truth 尚未完全收口”的 delete gate 与迁移目标。</task>
</tasks>

## Verification Criteria

- `core/telemetry/*` 已成为唯一 exporter formal home；exporter 只读 sources，不参与 runtime 编排；
- contracts 明确区分 `snapshot` 与 `views`，并制度化 `sensitivity` + `cardinality budget`；
- control plane 已有 `telemetry_surface` 作为唯一 bridge，后续 diagnostics/system-health 不再各自拼 truth；
- `DEPENDENCY_MATRIX/PUBLIC_SURFACES/VERIFICATION_MATRIX/FILE_MATRIX/RESIDUAL_LEDGER` 已同步登记 exporter family 与 delete gate；
- `tests/core/telemetry/*` 基础骨架已存在，能在后续接线时提供回归锚点。

## Must Haves

- single exporter formal home
- explicit contracts and ports
- credential-equivalent field policy (`password_hash`)
- cardinality budget rules
- governance sync before source hookup
