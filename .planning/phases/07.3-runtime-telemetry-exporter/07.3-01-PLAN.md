# Plan 07.3-01: 建立 RuntimeTelemetryExporter 的正式家园与执行边界

## Preconditions

已满足：
- `07.2` 已完成 architecture enforcement，observer-only hook 可继续扩展；
- `LiproProtocolFacade` 与 `Coordinator` 已分别是 protocol / runtime 正式 root；
- `07.3-CONTEXT.md` 已锁定 HA-only、pull-first、伪匿名化、真实时间戳、AI 调试友好等裁决。

## Tasks

1. **建立 formal telemetry home**
   - 新增 `custom_components/lipro/core/telemetry/`，落地 `models.py`、`ports.py`、`exporter.py`、`sinks.py`；
   - 在 `models.py` 固化 `TelemetrySnapshot`、`TelemetryViews`、`TelemetrySensitivity`、`CardinalityBudget`、`schema_version` 等稳定 contracts；
   - 明确单位、字段命名、真实时间戳策略与 `entry_ref` / `device_ref` 伪匿名引用规则，确保 schema 对 AI 调试与 replay 友好。

2. **锁定 source / exporter / control bridge 边界**
   - 在 `ports.py` 定义 `ProtocolTelemetrySource`、`RuntimeTelemetrySource`、`TelemetrySink` 等显式 ports；
   - 在 `exporter.py` 实现 `RuntimeTelemetryExporter`，只负责 pull sources、合并快照、执行 sensitivity/cardinality 预算、生成 sink views；
   - 新增 `custom_components/lipro/control/telemetry_surface.py` 作为 control-plane 唯一 bridge，禁止 diagnostics / system health 继续通过 `getattr` 或 runtime internals 漫游取数。

3. **建立 Wave 0 测试锚点**
   - 新增 `tests/core/telemetry/test_models.py`、`tests/core/telemetry/test_exporter.py`、`tests/core/telemetry/test_sinks.py`；
   - 覆盖稳定字段集合、真实时间戳保留、伪匿名引用规则、`password_hash`/token/secret 不出现在 views、cardinality budget 生效等基础断言；
   - 明确 exporter 只消费 protocol/runtime source 的快照与有界摘要，不允许测试依赖 compat shell 或 raw payload 穿透。

4. **同步治理真源**
   - 更新 `.planning/baseline/DEPENDENCY_MATRIX.md`，登记 `core/telemetry/*` 只能依赖 protocol/runtime sources，禁止反向依赖；
   - 更新 `.planning/baseline/PUBLIC_SURFACES.md` 与 `.planning/baseline/VERIFICATION_MATRIX.md`，明确 `RuntimeTelemetryExporter` / `TelemetrySurface` 的正式角色与 `OBS-01/OBS-02` 证据映射；
   - 更新 `.planning/reviews/FILE_MATRIX.md` 与 `.planning/reviews/RESIDUAL_LEDGER.md`，写清 exporter owner、plane 归属与“多口径 telemetry truth 收口” delete gate。

## Success Criteria

- `core/telemetry/*` 已成为 exporter 的唯一正式实现 home；
- exporter 明确是 pull-first observer，不具备 runtime 编排权，也不会演化为事件总线；
- contracts 已制度化真实时间戳、伪匿名引用、凭证等价物阻断与 cardinality budget；
- control plane 有且仅有 `telemetry_surface` 负责定位 exporter；
- baseline / residual 文档已登记 exporter family，后续 `07.4/07.5` 可直接复用其边界与证据。
