# Phase 25.2 Context

**Phase:** `25.2 Telemetry formal-surface closure and planning-truth sync`
**Milestone:** `v1.3 Quality-10 Remediation & Productization`
**Date:** `2026-03-17`
**Status:** `Ready for planning`
**Source:** `Phase 25` route freeze + `Phase 25.1` closeout + telemetry ghost-seam review

## Why Phase 25.2 Exists

当前 telemetry exporter / diagnostics / system health 的正式消费者已经建立，但 bridge helper 仍在 touched seam 上读取 `coordinator.client`。这会让已经完成的 `Coordinator.protocol` 正式收口在 control-plane observer seam 上被旧术语重新污染。

## Goal

1. 让 telemetry / diagnostics / system health consumer 只承认正式 `protocol` surface 或显式 port。
2. 让 `runtime_types.py`、bridge helper、测试夹具与 planning truth 对这条 seam 讲同一条诚实故事。
3. 在不新增第二条 root、不过度重做 exporter schema 的前提下，关闭 `coordinator.client` ghost seam。

## Decisions (Locked)

- touched seam 上**不保留** `client` fallback；formal-surface honesty 比兼容旧测试更优先。
- `RuntimeTelemetryExporter` 的 schema / sink payload 维持稳定；本 phase 只处理 source binding honesty 与 planning truth sync。
- `runtime_types.py` 只补 bridge 真正需要的最小正式 surface；不把整个 concrete coordinator 再次平铺成 mega protocol。
- `.planning/codebase/*` 仍只是 derived collaboration maps；若本 phase 触达，必须与 baseline / review / active truth 同步，但不能反向升级为 authority。

## Non-Negotiable Constraints

- 不得重新引入 `protocol + client` 双正式 surface，也不得发明 telemetry 第二 root。
- 不得顺手重做 telemetry schema、diagnostics payload 或 system-health projection。
- 不得新增测试文件；优先在现有测试文件中补回归，避免 toolchain-truth 漂移。
