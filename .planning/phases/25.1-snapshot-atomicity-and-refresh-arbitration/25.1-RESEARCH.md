# Phase 25.1 Research

**Status:** `research complete`
**Date:** `2026-03-17`
**Requirement:** `RUN-06`, `ERR-03`

## Executive Judgment

`25.1` 的核心不是“多补几个 if”，而是把 snapshot refresh 从 best-effort 改成 **atomic commit with explicit rejection semantics**。

最优拆分是 `3 plans / 3 waves`：

1. `25.1-01`：在 snapshot/device runtime 中建立 atomic rejection 语义与 last-known-good 保留规则
2. `25.1-02`：把 coordinator/update path 改成稳定失败语义，并修正现有回归测试
3. `25.1-03`：冻结 phase truth、closeout evidence 与 targeted validation

## Current Defect Chain

### 1. Pagination failure still publishes partial truth

`custom_components/lipro/core/coordinator/runtime/device/snapshot.py` 在分页抓取失败时仅记录错误并 `break`，随后继续构造 snapshot。这会让 page 1 成功、page 2 失败时，第一页被误当成新的全量 truth。

### 2. Parse failures are silently demoted to partial success

当前 builder 对设备级 parse error 仅 debug log 后 `continue`。这等于把“有问题的 full snapshot”降级成“看起来成功的 partial snapshot”。

### 3. Mesh topology enrich failure is swallowed

mesh-group topology enrich 当前失败后保留不完整快照。这同样违反 full snapshot atomicity。

### 4. Coordinator state publish has no arbitration boundary

`Coordinator._async_refresh_device_snapshot()` 默认信任 `DeviceRuntime.refresh_devices()` 返回值；因此 runtime 若返回 partial snapshot，coordinator 会无条件覆盖 `_state.devices`。

## Recommended Runtime Contract

### A. Atomic snapshot build

- fetch / parse / enrich 任一关键环节失败时，builder 必须拒绝生成新 snapshot。
- 拒绝失败应有稳定异常类型，至少包含 stage / page / cause-type 等可测试字段。

### B. Explicit last-known-good retention

- `DeviceRuntime` 仅在完整成功时更新 `_last_snapshot`。
- 若已有 `_last_snapshot` 且本轮 refresh 被拒绝，旧快照继续保留，但 refresh 调用本身必须失败。

### C. Coordinator-level failure surfacing

- `Coordinator._async_update_data()` 必须把 snapshot rejection 走进 `UpdateFailed` / telemetry failure summary 主链。
- `Coordinator.async_refresh_devices()` 也必须遵循相同语义：失败就失败，不发布 partial state。

### High-risk scenarios that must be locked

- page 2 fetch fails after page 1 success
- malformed `devices` payload or non-dict row appears in page payload
- one device parse fails in an otherwise valid page
- mesh group topology enrich fails after device rows fetched
- refresh fails after one prior successful snapshot; old snapshot must remain available and state must not be overwritten
