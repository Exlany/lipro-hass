<objective>
提炼并固定 `auth` / `device` 方向已经成熟的 host-neutral contracts，与 HA adapter 侧的 login/bootstrap/platform projection seams 明确分层：让可复用 truth 不再散落在 `config_flow.py`、`entry_auth.py`、`flow/login.py` 与 `core/device` 的 HA 平台语义之中，同时不改变 `LiproProtocolFacade` / `Coordinator` 的正式根身份。
</objective>

</tasks>

## Verification Criteria

- `AuthSessionSnapshot` / login result truth 只剩一条正式 host-neutral contract story，HA flow 不再并行定义第二套 canonical result；
- `config_flow.py` / `entry_auth.py` 的 reusable auth/bootstrap semantics 已被抽成显式 seam，HA adapter 只保留输入、持久化与异常映射职责；
- `core/capability` / `core/device` 不再把 Home Assistant platform strings 当成 host-neutral capability truth 的组成部分；
- `entry_auth.py` outward seam 仍保持稳定，不因 inward-move 破坏 controller wiring 与测试 patch 入口；
- `LiproProtocolFacade` / `Coordinator` / `coordinator_entry.py` 未被重写为第二条 boot story 或新的 shared-core root。

## Must-Haves

- 不得引入第二个 runtime root、第二个 protocol façade 或 generic host framework
- 不得在 `core/auth`、`core/device`、`core/capability` 中新增 `homeassistant` imports
- 不得把 `config_flow.py` / `entry_auth.py` 改回 raw vendor payload / raw login dict consumer
- 不得通过大规模 rename churn 掩盖真实 seam；优先做 contract/home 收口
