</tasks>

## Verification Criteria

- 成熟的 auth/bootstrap helpers 已进入 host-neutral home，`flow/login.py` / `entry_auth.py` 不再承担 formal reusable truth 的主存放地；
- device/capability nucleus 与 HA platform projection 已完成物理分层，平台模块通过显式 adapter seam 选设备，而不是要求 core truth 直接携带 HA 平台枚举/字符串；
- 如存在 transitional helper / alias，均具备明确 delete gate，且没有新增 silent residual；
- 平台装配回归保持稳定，没有因 projection demotion 打破 entity setup 或 device filtering 行为。

## Must-Haves

- 不得把 `helpers/platform.py` 或相邻 adapter helper 演化成新的业务根
- 不得把 headless/CLI consumer proof 偷跑进本 plan
- 不得做无必要的大规模 package split 或 public export 扩面
- 任何 transitional projection 都必须比当前事实更窄、更显式、更容易删除
